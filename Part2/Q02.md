# Part2 | 1万個のパーティクルをInstancedMeshで60fps維持したまま動かそう

## 目標

Draw Callがボトルネックになる典型例を体験し、InstancedMeshを使ってDraw Call数を劇的に減らす最適化ができるようになる。

## 今回の演習で学ぶこと

### 描画コストの正体

3Dシーンが重くなる原因は主に3つあります。

- **Draw Call数**: GPUに「この形を描いて」と指示を送る回数。回数が多いほどCPU側のオーバーヘッドが増える
- **ポリゴン数**: 頂点・面の数。GPUの処理量に直結する
- **オーバードロー**: 同じピクセルが何度も塗り直される現象

現代のGPUは「1回あたりの描画」自体は高速です。問題になりやすいのは、**同じような物を大量に個別描画してDraw Call数が積み上がる**ケースです。今回の「1万個の箱」はまさにこのパターンです。

### なぜMeshを1万個生成すると重いのか

`new THREE.Mesh(geometry, material)` を1万回呼び出すと、ジオメトリとマテリアルは使い回していても、Three.jsは1万個の**別々のオブジェクト**として扱い、`renderer.render()` のたびに1万回のDraw Callを発行します。CPUが「次はこれを描いて」とGPUに指示を出す処理には無視できないオーバーヘッドがあり、これが積み重なるとフレームレートが大きく低下します。

### InstancedMeshという解決策

`THREE.InstancedMesh` は、同じジオメトリ・マテリアルを持つオブジェクトを「1つのDraw Callで、位置や回転だけを変えて大量に複製描画する」仕組みです。

- インスタンスごとの位置・回転・スケールは「4x4の変換行列」として`setMatrixAt(index, matrix)`で書き込む
- 一度書き込んだら`instanceMatrix.needsUpdate = true`を立てることでGPUに反映される
- 位置が変わらないインスタンスであれば、**初期化時に1回だけ**書き込めばよく、毎フレーム更新する必要はない

### 計測してから最適化する、という姿勢

「なんとなく重い」で終わらせず、必ず数値で確認します。今回はstats.jsを使い、画面左上に表示されるFPSパネルで、最適化前後の変化を見比べます。感覚ではなく数値で語れることが、実務での最適化提案の説得力になります。

## 問題文

配布された `src/main.ts` は、1万個のBoxMeshを個別に生成してシーンに追加する実装です。動作はしますが、Draw Callが1万回発生しており、フレームレートが低下しています。

1. まず現状のFPSをstats.jsパネルで確認し、記録してください(Before計測)
2. `THREE.InstancedMesh` を使い、1万個の箱を1回のDraw Callで描画できるように書き換えてください
3. インスタンスごとの位置はランダムに配置し、見た目が現状の実装と同程度になるようにしてください
4. 書き換え後のFPSを再度記録し、Before/Afterを比較してください(After計測)

### 完成イメージ

```
- 画面に1万個の小さな箱がランダムに配置され、全体がゆっくり回転している
- 見た目はリファクタリング前とほぼ変わらない
- stats.jsパネルのFPSが、リファクタリング前より大幅に改善している
```

## ヒント

- `THREE.InstancedMesh(geometry, material, count)` のコンストラクタで、最大インスタンス数`count`を指定します
- `THREE.Object3D` を「計算用のダミーオブジェクト」として1つ用意し、`dummy.position.set(...)` → `dummy.updateMatrix()` → `mesh.setMatrixAt(i, dummy.matrix)` の順で書き込むと、位置計算のコードを使い回せます
- 全インスタンスを一括で回転させたい場合は、インスタンスごとの行列を毎フレーム更新するのではなく、`InstancedMesh`自体(親オブジェクト)の`rotation`を変更するだけで済みます
- `renderer.info.render.calls` をコンソールに出力すると、Draw Call数を直接確認できます

## 達成基準

- [ ] `THREE.InstancedMesh` を使い、1万個のオブジェクトが1つのDraw Callで描画されている
- [ ] 最適化前後のFPSを両方計測し、比較できる状態になっている
- [ ] インスタンスごとの位置がランダムに分布し、見た目が最適化前と同程度になっている
- [ ] `any` を使わず、型が明示されている

## よくあるエラー

- `setMatrixAt` を呼んだ後に `instanceMatrix.needsUpdate = true` を忘れ、画面に反映されない → 必ずループの後(または各回)に立てる
- `InstancedMesh` のコンストラクタに渡す `count` と、実際に `setMatrixAt` でループする回数が一致していない → 定数化して揃える
- 毎フレーム全インスタンスの行列を再計算してしまい、期待したほどFPSが改善しない → 静的な配置であれば初期化時の1回だけで十分

import * as THREE from 'three';

export interface SceneContext {
  scene: THREE.Scene;
  camera: THREE.PerspectiveCamera;
  renderer: THREE.WebGLRenderer;
}

export function createScene(container: HTMLElement): SceneContext {
  const scene = new THREE.Scene();

  const camera = new THREE.PerspectiveCamera(
    60,
    window.innerWidth / window.innerHeight,
    0.1,
    200
  );
  camera.position.set(0, 0, 60);

  const directionalLight = new THREE.DirectionalLight(0xffffff, 1);
  directionalLight.position.set(5, 5, 5);
  scene.add(directionalLight);
  scene.add(new THREE.AmbientLight(0xffffff, 0.4));

  const renderer = new THREE.WebGLRenderer({ antialias: true });
  renderer.setSize(window.innerWidth, window.innerHeight);
  renderer.setPixelRatio(window.devicePixelRatio);
  container.appendChild(renderer.domElement);

  window.addEventListener('resize', () => {
    camera.aspect = window.innerWidth / window.innerHeight;
    camera.updateProjectionMatrix();
    renderer.setSize(window.innerWidth, window.innerHeight);
  });

  return { scene, camera, renderer };
}

import * as THREE from 'three';

export interface ParticleFieldOptions {
  count: number;
  spread: number;
}

export class ParticleField {
  readonly mesh: THREE.InstancedMesh<THREE.BoxGeometry, THREE.MeshStandardMaterial>;

  constructor({ count, spread }: ParticleFieldOptions) {
    const geometry = new THREE.BoxGeometry(0.3, 0.3, 0.3);
    const material = new THREE.MeshStandardMaterial({ color: 0x4f8fdb });

    this.mesh = new THREE.InstancedMesh(geometry, material, count);

    const dummy = new THREE.Object3D();
    for (let i = 0; i < count; i++) {
      dummy.position.set(
        (Math.random() - 0.5) * spread,
        (Math.random() - 0.5) * spread,
        (Math.random() - 0.5) * spread
      );
      dummy.updateMatrix();
      this.mesh.setMatrixAt(i, dummy.matrix);
    }

    this.mesh.instanceMatrix.needsUpdate = true;
  }

  tick(rotationDelta: number): void {
    this.mesh.rotation.y += rotationDelta;
  }
}

import '../style.css';
import { createScene } from './scene/createScene';
import { ParticleField } from './entities/ParticleField';
import { createStatsPanel } from './utils/perf';

const appElement = document.getElementById('app');
if (!appElement) {
  throw new Error('#app element not found');
}

const { scene, camera, renderer } = createScene(appElement);
const stats = createStatsPanel();

const particleField = new ParticleField({ count: 10000, spread: 80 });
scene.add(particleField.mesh);

function animate(): void {
  stats.begin();
  requestAnimationFrame(animate);

  particleField.tick(0.002);

  renderer.render(scene, camera);
  stats.end();
}

animate();
