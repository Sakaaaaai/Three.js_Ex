# Part7 | モデルの格納方式・テクスチャ解像度による圧縮効果を比較しよう

## 目標

GLTFLoader・DRACOLoader・KTX2Loaderを使ったモデル読み込みの基本構成を理解し、アセットの持ち方(ファイル分割・テクスチャ解像度)によってファイルサイズと読み込み時間がどう変わるかを、実際に計測して比較できるようになる。

## 今回の演習で学ぶこと

### GLTFフォーマットの基礎(.gltf と .glb)

- `.gltf`: JSON形式のシーン記述ファイル。頂点データ(`.bin`)やテクスチャ(`.png`など)は別ファイルに分かれることが多い
- `.glb`: JSON・バイナリデータ・画像をすべて1つのファイルにまとめたバイナリ形式

`.gltf`(分割)は人間が中身を読みやすい・デバッグしやすい一方、ブラウザから読み込む際に複数のHTTPリクエストが発生します。`.glb`は1リクエストで完結するため、特にリクエスト数自体がボトルネックになりやすい環境(モバイル回線など)で有利です。

### テクスチャ解像度の見直しという基本的だが見落とされがちな最適化

3Dモデルのファイルサイズは、ジオメトリよりもテクスチャが大部分を占めることがよくあります(このサンプルは比較のためにジオメトリを主体にしていますが、実務のモデルではテクスチャの影響がさらに大きくなります)。「本当にその解像度が必要か」を見直すだけで、Draco/KTX2のような専用の圧縮技術を使わなくても、シンプルにファイルサイズを削減できます。

### DracoLoader・KTX2Loaderの役割

`GLTFLoader`は、標準の(圧縮されていない)glTFに加えて、拡張仕様を使って圧縮されたモデルも読み込めます。

- `DRACOLoader`: ジオメトリ(頂点データ)がDraco圧縮されている場合に必要なデコーダー
- `KTX2Loader`: テクスチャがKTX2(Basis Universal)形式で圧縮されている場合に必要なトランスコーダー

これらのデコーダーはWASM(WebAssembly)ファイルとして提供されるため、`public/draco/`・`public/basis/`にあらかじめ配置し、`setDecoderPath`/`setTranscoderPath`でその場所を教えてあげる必要があります(このプロジェクトにはすでに`three`パッケージ付属のデコーダーファイルをコピーしてあります)。

今回配布しているサンプルモデルはDraco/KTX2で圧縮されていませんが、ローダー自体はいつでも圧縮モデルを読み込めるように構成しておくのが実務では一般的です。より発展的な学習として、この文末の「発展: 実際にDraco/KTX2で圧縮してみる」も参照してください。

## ヒント

- `DRACOLoader`・`KTX2Loader`のデコーダーパスは、このプロジェクトでは`/draco/`・`/basis/`です(`public/draco/`・`public/basis/`が、ビルド時にそのままルートパスとして配信されます)
- `KTX2Loader`の`detectSupport(renderer)`は、使用しているGPU・ブラウザがどのテクスチャ圧縮形式に対応しているかを判定するために必要です。呼び忘れると、KTX2テクスチャを含むモデルの読み込みでエラーになります
- 読み込み時間の計測は`performance.now()`を使い、`await loader.loadAsync(url)`の前後の差分を取ります
- ファイルサイズの計測は、`fetch(url).then(res => res.arrayBuffer())`で取得したデータの`byteLength`を使うと、実際に転送されたバイト数を正確に取得できます。`.gltf`分割形式の場合は、`.gltf`・`.bin`・テクスチャの3ファイル分を合計する必要があります

## 達成基準

- [ ] `GLTFLoader`に`DRACOLoader`・`KTX2Loader`が正しくアタッチされている

- [ ] `any`を使わず、型が明示されている

## よくあるエラー

- `DRACOLoader`の`setDecoderPath`に末尾スラッシュを付け忘れる(`/draco`ではなく`/draco/`が必要な場合がある)
- `KTX2Loader`の`detectSupport(renderer)`を呼び忘れ、KTX2テクスチャを含むモデルで実行時エラーになる
- `.gltf`(分割形式)のファイルサイズ計測で、`.bin`やテクスチャの分を合計し忘れ、実際より小さいサイズとして計測してしまう
- `fetch`のキャッシュにより、2回目以降の計測でファイルサイズは合っていても読み込み時間が極端に速くなる(ブラウザキャッシュの影響)。フェアに比較したい場合は、DevToolsの「Disable cache」を有効にすると良い

## 発展: 実際にDraco/KTX2で圧縮してみる

今回のサンプルはあえて圧縮していない状態で比較していますが、[gltf-transform](https://gltf-transform.dev/)というCLIツールを使うと、手元で実際にDraco圧縮・KTX2圧縮を試すことができます。

```bash
# gltf-transform CLIをインストール(このプロジェクトにはインストールされていません)
npm install --no-save @gltf-transform/cli

# ジオメトリをDraco圧縮
npx gltf-transform draco public/models/sample-glb-large-tex/model.glb public/models/model-draco.glb

# テクスチャをKTX2(Basis Universal)圧縮
npx gltf-transform ktx-basisu public/models/sample-glb-large-tex/model.glb public/models/model-ktx2.glb
```

生成された`model-draco.glb`・`model-ktx2.glb`を`public/models/`に置き、`src/main.ts`の`variants`配列にエントリを追加すれば、このハーネスでそのまま比較できます(すでに`DRACOLoader`/`KTX2Loader`をセットアップしてあるので、追加の実装は不要です)。

```ts
import * as THREE from 'three';
import { GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js';
import { DRACOLoader } from 'three/examples/jsm/loaders/DRACOLoader.js';
import { KTX2Loader } from 'three/examples/jsm/loaders/KTX2Loader.js';

export function createGltfLoader(renderer: THREE.WebGLRenderer): GLTFLoader {
  const dracoLoader = new DRACOLoader();
  dracoLoader.setDecoderPath('/draco/');

  const ktx2Loader = new KTX2Loader();
  ktx2Loader.setTranscoderPath('/basis/');
  ktx2Loader.detectSupport(renderer);

  const gltfLoader = new GLTFLoader();
  gltfLoader.setDRACOLoader(dracoLoader);
  gltfLoader.setKTX2Loader(ktx2Loader);

  return gltfLoader;
}
```

```ts
import type { GLTF, GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js';

export interface ModelVariant {
  label: string;
  entryUrl: string;
  assetUrls: string[];
}

export interface MeasurementResult {
  variant: ModelVariant;
  gltf: GLTF;
  loadTimeMs: number;
  totalBytes: number;
}

async function fetchTotalBytes(urls: string[]): Promise<number> {
  const buffers = await Promise.all(urls.map((url) => fetch(url).then((res) => res.arrayBuffer())));
  return buffers.reduce((sum, buffer) => sum + buffer.byteLength, 0);
}

export async function measureVariant(
  loader: GLTFLoader,
  variant: ModelVariant
): Promise<MeasurementResult> {
  const start = performance.now();
  const gltf = await loader.loadAsync(variant.entryUrl);
  const loadTimeMs = performance.now() - start;

  const totalBytes = await fetchTotalBytes(variant.assetUrls);

  return { variant, gltf, loadTimeMs, totalBytes };
}
```

```ts
const variants: ModelVariant[] = [

    entryUrl: '/models/sample-separate/model.gltf',

      '/models/sample-separate/model.gltf',
      '/models/sample-separate/model.bin',

    label: '.glb(PNG 512x512 埋め込み)',

  100

camera.position.set(0, 0.6, 4);

function renderReport(results: MeasurementResult[]): void {
  if (!reportElement) return;
  const lines = results.map((result) => {

async function main(): Promise<void> {

main().catch((error) => {
```
